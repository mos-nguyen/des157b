Feasibility Report

Features

  • A home page that has the option to start, go into settings, and return to the portal page
  • Sound and animated rain to add to atmosphere/theme of the game
  • custom cursor for atmosphere/theme
  • Animated text/dialogue that appears like its being typed on a type writer
  • An inventory space
  • Interactable items on the screen that users can click to add to their inventory
  • A way to combine items and information to uncover more about the mystery
  • A "health bar" that measures the amount of tolerance left
  • A way to escape/close out/start over so user isn't required to finish the entire game to be done - needs an alert saying progress wont be saved -

Technologies Required

  • Overlays
  • Sound JS Library
  • Particles JS Lirbary
  • GSAP text JS Library
  • CSS Positioning
  • Custom Cursor JS Library
  • Draggable.JS

Skill Assessment

For features like the starting page, adding sound, and creating a health bar are all things that I am capable of doing. Many of these features are similar to the 157A Project Game On! Which I plan on reviewing in order to bring this project to life. As for the libraries, the main one I want to use for my project is draggable.js which, if similar to the J Query examples in class, then I believe I will be able to become skilled within the given timeframe. As for the other libraries, I believe I will be able to learn about them and be able to use them within the time frame. However, if I cannot then it won't affect the interactiveness of the project, just the atmosphere/theme. Much of the heavy lifting will involve reviewing past projects.

Difficulties

Time wise, I think the illustrated assets and typing out a cohesive themed story will take up a lot since I will have to do a lot of brainstorming and sketching before getting to the final. I plan on having roughly/quickly made sketches as placeholders so I can focus on the HTML, CSS, and JS first. This way I can work on the illustrations thoughtout the weeks and replace each asset as I finish them.

As I was planning out the interactions for this project, I've realized i've pushed the topic of censorship to the side in order to focus more on the theme of the game. I don't want this to be the case so I will have to revisit my proposal and brainstorm to keep the topic of censorship in the forefront of my mind. I'm hoping that doing this will allow my project to revolve around censorship rather than trying to shoehorn it in.
