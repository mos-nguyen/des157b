Capstone Usability Testing Results

Carol Nguyen
Testers: Tegan Leong, Johnson Ou, Elysia Toumai

Return to Portal Button

Pretty much everyone, after finishing the listed tasks, started clicking around and eventually clicked the door icon. Each time the user was surprised/confused. I plan to add some way for players to know what each icon does so that there is no longer any confusion. Some options I thought about while talking with the players include:

  • Having a pop up note on screen that explains it when players hover over the icon
  • Having an alert when players click on the Return to Portal button so they can back out
  • Adding short labels underneath the icons
  • Separating the Return to Portal button from the rest of the interface so its not accidentally clicked

Speaking Characters

While everyone knew which character was speaking when, both the players and I agreed there could be ways to make it clearer. Elysia brought up that it would be important once I add other characters like the two other witnesses. The other witnesses will have the same role and the designs wont be as strong as the detective fox wearing a sterotypical detectives coat. Some options I thought about while talking with the players include:

  • Dimming the character that isn't speaking at the moment
  • Having the speaking character move/be animated when they start talking

Ideally, Id like to implement all the features to make it more visually interesting, but for now I will try to figure out dimming the characters. I will also need to figure out how to make the text box go back and forth between different text banks for the different characters.

Viewport

For some reason, when viewing the game on laptops the viewport will sometimes break and causes scrolling. Smaller laptops result in certain buttons getting cut off which was an issue for Johnson and made the game a little confusing. In case I cant figure it out, I may add the buttons closer to the top or middle so theres no risk of it being cut off completely.

Conclusion

Overall, users could figure out how to navigate and interact with my project with minimal confusion. Right now, the message of censorship has not been added, so the project hasnt impacted anyone the way i indented it to yet. Im hoping once I add the reputation bar, scramble text, and uncensoring feature that the message/theme will be clear. Along with the modifications above, I believe adding more red within the instructions/text will make it stronger. I want the entire playing experience to be immersive. I will see if working on a smaller screen can help me figure out the viewport. I plan to continue on getting the entire narrative and interactions implemented to my project which will require me to go back and forth between writing the story, designing the characters, and working on the code. If I manage to finish with the required parts early, then I will work on features that will improve the overall aesthetic as well as trying to implement sound.
